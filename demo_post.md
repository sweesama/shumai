## 1. Demo 简介

**是什么**：书脉 AI 是一个基于 AI 的古籍阅读与知识探索网站（Web 应用）。用户可以阅读 83 部中国经典古籍原文，通过 AI 一键生成知识图谱，并对任意文言文段落进行智能分析与翻译。

**面向谁**：
- 中学/大学文言文学习者——需要原文阅读 + 白话翻译 + 知识梳理
- 中国传统文化爱好者——希望以非线性的方式探索古籍中的人物、地点、概念关系
- 数字人文研究者——需要快速提取文本实体并可视化知识结构

**主要功能**：

**功能一：古籍书架 + 原文阅读**
收录 83 部经典古籍（论语、庄子、山海经、史记等），支持全文阅读、章节切换、字体调节。首页精选《山海经》展示 AI 活化效果。

> 📸 截图位置：首页书架 + 阅读器界面（需插入截图）

**功能二：AI 知识图谱活化**
选择任意古籍点击「AI 活化」，DeepSeek 大模型自动提取文本中的人物、地点、概念等实体，生成可交互的力导向知识图谱。支持节点拖拽、缩放、点击查看详情。流式生成，实时展示进度。

> 📸 截图位置：知识图谱界面（需插入截图）

**功能三：AI 文本分析**
粘贴任意文言文段落，AI 自动进行：实体提取（人名/地名/概念）、白话文翻译、深度解读。左右双栏布局，左侧输入右侧结果，支持载入示例一键体验。

## 2. Demo 创作思路

**灵感来源**：
阅读古籍时最大的痛点不是"找不到书"，而是"读不懂、记不住、看不到关联"。传统的古籍网站只提供原文扫描件或纯文本，读完一段后，里面提到的人物、地名、概念之间的关系完全靠脑补。我想：如果 AI 能自动帮我把一本书里所有人物、地点、概念提取出来，画成一张知识关系网，那阅读体验会完全不同。

**想解决的问题**：
- **读不懂**：文言文门槛高，普通读者需要翻译和注释
- **记不住**：一本书几十个人物、几十个地名，读到后面忘了前面
- **看不到关联**：传统线性阅读无法呈现知识的网状结构

**为什么做这个方向**：
1. LLM 的实体提取和关系推理能力非常适合处理古籍文本——人物、地点、事件关系明确
2. 知识图谱的可视化能将"线性阅读"升级为"网状探索"，这是传统古籍工具做不到的
3. 市面上的古籍 App 要么是纯文本库（国学导航），要么是扫描件（中华典籍网），没有 AI 活化能力
4. DeepSeek API 成本低、中文理解能力强，适合做古籍这种垂直领域的 NLP 任务

## 4. TRAE 实践过程

### 4.1 项目架构

- **前端**：单文件 HTML（原生 JS + CSS，无框架），包含书架、知识图谱（D3.js 力导向图）、阅读器、文本分析、笔记五个模块
- **后端**：Python Flask，提供古籍文本 API + DeepSeek AI 活化接口（SSE 流式输出）+ 文本分析接口
- **数据**：83 本古籍纯文本（texts/ 目录），9 本预置 AI 活化缓存（cache/ 目录）
- **部署**：Railway（Flask 长连接 + 持久化 Volume），GitHub 自动部署

### 4.3 踩坑经历

1. **Vercel 部署失败**：Flask 的 SSE 长连接在 Vercel Serverless 上超时，迁移到 Railway 解决
2. **浏览器缓存**：改了代码但用户看不到变化，在 server.py 加 no-cache 响应头解决
3. **封面图裁剪**：竖版封面图用 background-size:cover 在横版容器中被裁剪，改为 img + object-fit:contain 彻底解决
4. **卡片不可见 bug**：IntersectionObserver 只绑定一次，renderShelf 重渲染后新卡片 opacity:0 永不显示，改为 setTimeout 错峰显示
5. **知识图谱切页中断**：SSE 生成中切页后进度遮罩消失，加 state.isGenerating 标志切回时恢复
